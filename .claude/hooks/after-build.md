# After every code change — learning log

After completing any code change (new feature, refactor, bug fix), you must:

1. **Append an entry to `LEARNINGS.md`** using this structure:

```
## [YYYY-MM-DD] — <what was built, one line>

**What changed**
- bullet list of files touched and what each does now

**How it connects**
- how this fits into the existing architecture (data flow, which components/routes/modules interact)

**New concepts**
- any React, TypeScript, Node, Express, or MongoDB concept that appeared for the first time, explained briefly with a Vue/JS analogy where relevant
```

2. **Explain it in chat** — after writing the log entry, give the same breakdown in your response so Sebastian can read it without opening the file.
